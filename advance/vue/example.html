Classic Case ​

路由前置守卫 ​

在前端应用中，如果使用 localStorage 或 sessionStorage 来存储由 jsonwebtoken 生成的令牌（token），当该令牌过期时，可以使用导航守卫（navigation guards）来在每次路由跳转前检查令牌是否过期。

ts
interface DecodedToken{
    iss: number
    sub:number
    aud:number
    exp: number
    iat: number
}

router.beforeEach((to, from, next) => {
  const authStore = useAuthStore()
  const { auth } = storeToRefs(authStore)

  try {
    // 避免无限循环：如果已经在登录页，无需再次重定向
    if (to.path === '/login' && !auth.value) {
      return next()
    }

    if (auth.value) {
      const decodedToken = jwtDecode(auth.value) as DecodedToken
      const currentTime = Date.now() / 1000
      if (decodedToken.exp < currentTime) {
        auth.value = ''
        if (to.path !== '/login') {
          return next('/login')
        }
      } else if (to.path === '/login') {
        return next('/admin')
      }
    } else if (to.path !== '/login') {
      return next('/login')
    }
    
    return next()
  } catch (e) {
    console.error("JWT 解码失败：", e)
    if (to.path !== '/login') {
      return next('/login')
    }
    return next()
  }
})

TIP

jwtDecode 是一个JavaScript库，用于解码 JSON Web Token（JWT）字符串，将其转换为一个 JavaScript 对象，以便您可以访问存储在其中的各种字段和属性。需要注意的是，这个库只做解码操作，它不会验证签名的有效性。

  1. 安装
bash
npm install jwt-decode
  1. 用法
js
import jwtDecode from 'jwt-decode';

const token = "你的JWT令牌";
const decodedToken = jwtDecode(token);

console.log(decodedToken);

解码后的对象会包含 JWT 的 payload 部分，通常这会是一个包含各种声明（claims）的对象。例如，标准的 JWT 声明可能包括：

  • iss: 发行人
  • sub: 主题
  • aud: 受众
  • exp: 过期时间
  • iat: 发行时间

以及可能还有其他自定义的声明。
